# 쇼핑행동분석

## 상품별 보고서
쇼핑행동분석의 상품별 리포트에서는 상품을 결제할 때 발생하는 고객들의 주요 행동들에 대해 분석할 수 있습니다.

### 차원

차원 | 설명 
-|-
상품 카테고리 (대) | 스마트스토어에서 상품 등록 시 선택한 (대) 카테고리 입니다.
상품 카테고리 (중) | 스마트스토어에서 상품 등록 시 선택한 (중) 카테고리 입니다.
상품 카테고리 (소) | 스마트스토어에서 상품 등록 시 선택한 (소) 카테고리 입니다.
상품명 | 스마트스토어에서 상품 등록 시 입력한 상품명 입니다.
상품ID | 스마트스토어에서 상품 등록 시 부여받은 고유의 상품번호 입니다.
배송유형 | 해당 제품의 보고서 생성 시 설정된 배송유형 입니다.  일반배송, 오늘출발, N배송(구, 네이버도착보장), N판매자배송(구, 판매자도착보장), N희망일배송(구, 판매자희망일도착보장) 5가지가 있습니다.

### 지표

지표그룹 | 지표 | 설명
-|-|-
행동 | 상품 상세 조회수 | 상품 상세보기 페이지의 조회수를 집계합니다.
결제 | 결제금액 | 상품 할인 금액, 배송비 등을 제외하고 실제로 결제된 금액입니다.  ('매출'과 동일)
결제 | 결제 상품수량 | 판매된 상품 총 수량을 나타냅니다.
결제 | 상세조회 대비 결제율 | 상세 조회 당 결제한 상품수량을 계산하여 결제 확률을 보여줍니다.  (= 결제된 상품 수량/상품상세 조회수)


## 페이지별 보고서

- 페이지별 보고서에서는 [각 페이지 URL 별, 페이지 수, 체류시간, 평균 체류시간] 정보를 제공합니다. 
- 어떤 페이지가 얼마나 조회되고, 해당 페이지에 얼마나 머무는지 정보를 이용하여 페이지에 대한 이용자의 반응을 알 수 있습니다.
- 보고서에 보여주는 [페이지 URL]의 과도한 파편화(줄이 많아지는 현상)를 막기 위해 URL 뒤에 붙어 있는 parameter와 value를 제외한 뒤에 동일해지는 URL로 묶어서 정보를 제공합니다.
- 동일한 host 정보가 화면을 차지하고 있는 것을 방지하기 위해 URL에서 host 부분은 제외하고 제공합니다.

### 예시
-  실제 URL이 [ https://www.abc.com/index.html?product_id=1&category=2 ]인 경우 실제 URL에서 아래 부분은 제외합니다.
  -·[https://], [http://] 제외
· - host 부분인 [www.abac.com] 제외
  - host 부분인 [www.abac.com] 제외
  -·parameter와 value 부분인 [?product_id=1&category=2] 제외 남은 부분인 [/index.html]를 페이지 URL 로 제공합니다.

### 차원
- 페이지 URL : 이용자가 방문한 사이트의 URL입니다. host와 parameter와 value를 제외하고 제공합니다.


### 지표

지표 | 설명
-|-
페이지수 | 이용자가 해당 페이지 URL를 조회한 횟수입니다.
체류시간 | 각 페이지 URL에 이용자들이 체류한 시간의 합계 입니다.
평균 체류시간 | 이용자가 해당 페이지 URL에 머문 평균 시간입니다.  (= 체류시간/페이지 수)


## 상품 상세 조회수가 '0'으로 노출되는 경우
- 상품 상세 조회 수가 0인데 결제 금액과 결제 상품 수량이 0이 아닌 경우는 추가 구성 상품이 원 상품과 함께 결제되었을 때입니다. 
- 해당 상품 상세페이지에서 발생한 조회 수는 원 상품 기준으로 잡히기 때문에 추가 구성 상품의 조회 수는 별도로 집계되지 않고 있습니다.

## FAQ
1. 모든 리포트에서 '간단' 모드의 집계 수치가 '상세' 모드에서 나타나지 않는 경우가 있습니다. 이는 '상세'모드의 데이터 표시가 1,000 개로 제한되어 발생할 수 있습니다. 엑셀 다운로드를 이용하면 모든 데이터를 확인할 수 있습니다.
